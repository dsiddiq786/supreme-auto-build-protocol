# Contributing to Supreme Auto-Build Protocol

Thank you for your interest in contributing to the Supreme Auto-Build Protocol!

## Code of Conduct

- Be respectful and inclusive
- Follow professional coding standards
- Document all changes
- Test thoroughly before submitting

## Development Process

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
4. **Test your changes**
   ```bash
   npm test
   npm run lint
   ```
5. **Commit with conventional commits**
   ```bash
   git commit -m "feat: add new AI model integration"
   ```
6. **Push and create a Pull Request**

## Commit Message Guidelines

- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation changes
- `style:` Code style changes
- `refactor:` Code refactoring
- `test:` Test additions/changes
- `chore:` Build process or auxiliary tool changes

## Pull Request Process

1. Update the README.md with details of changes
2. Update the CHANGELOG.md
3. Ensure all tests pass
4. Request review from maintainers
